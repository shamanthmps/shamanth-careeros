# SDLC

## Executive Summary
SDLC (Software Development Lifecycle) provides the end-to-end framework from requirements to maintenance. Interviewers expect TPMs to explain how SDLC stages integrate with quality, security, and release governance.

Strong answers show lifecycle tailoring by product risk and business urgency.

## Why This Matters
- TPM relevance: SDLC discipline reduces rework and late defects.
- Architecture relevance: Design quality and testability are set early in SDLC.
- Interview relevance: Common execution-depth topic.

## Core Concepts
### Definitions
- SDLC: Structured stages of software delivery.
- Stage gate: Quality checkpoint before moving forward.
- Shift-left: Earlier validation of quality/security concerns.

### Internal Working
Requirements -> Design -> Build -> Test -> Deploy -> Operate -> Improve

### ASCII Diagram
Idea -> PRD -> Design -> Code -> QA -> Release -> Monitoring

### Trade-offs
- Rigid stage gates vs agility.
- Upfront planning vs iterative learning.

## Real-world Example
A platform team introduced architecture and test-readiness gates before implementation. Defect leakage reduced and release confidence improved.

## TPM Perspective
Ask:
- Which SDLC stages are currently weak?
- What gate criteria are objective and measurable?
- How are cross-team dependencies tracked?

## Interview Perspective
Strong answer:
"I tailor SDLC by risk: critical systems get stricter gates, low-risk features get lighter governance, but all paths maintain traceability and quality feedback loops."

## Common Mistakes
- One-size-fits-all lifecycle.
- Gates without objective criteria.
- No operational feedback into planning.

## 5-Minute Revision
- SDLC structures delivery quality.
- Shift-left reduces downstream cost.
- Risk-tiered governance is best practice.

## Practice Interview Questions
1. How do you modernize legacy SDLC?
2. What should be in a release-readiness gate?
3. How do you implement shift-left effectively?
4. How should SDLC differ for regulated systems?
5. What SDLC metrics do executives care about?

## Key Takeaways
- SDLC quality determines delivery quality.
- TPMs should enforce measurable stage criteria.
- Lifecycle must be risk-aware and adaptive.
